# /resume

`/resume` 用来恢复你之前保存过的会话。

当你退出 Hunea 后重新打开，想接着上次的对话继续聊，或者在当前会话里想切回某个更早保存过的对话，都可以用它。输入 `/resume` 确认后，会打开一个全屏的会话选择面板，标题是 `Resume Session`，你挑选想要恢复的那条会话即可。

## 列表概览

面板会列出**当前工作区/项目**下此前保存过的会话，并且**不会包含当前正在进行的这条会话**（毕竟你已经在里面了）。

每一条大致会展示三行信息：

1. 该会话里**第一条用户消息**的摘要（如果还没有用户消息，则回退到会话标题）
2. **最近一条助手回复**的摘要（同理，没有则回退到标题）
3. 元信息：相对时间（比如 `now`、`5m`、`2h`、`1d`）· 工作目录 · 会话体积（有的话）

列表加载时可能会先闪一下 `Loading sessions...`；如果当前项目下还没有可恢复的会话，则会显示 `No sessions`。

> 会话数据会落在 Hunea 的数据目录里（一般是 `~/.config/hunea/`）。不过 `/resume` 本身是按**当前打开的工作区**来筛的，所以换到另一个项目目录启动时，列表内容也会跟着变。

## 怎么操作

底部会有一行快捷键提示，常用操作如下：

- `↑` / `↓`，或者 `j` / `k`：上下移动选中项
- `←` / `→`，或者 `h` / `l`：按页翻动（会话多的时候比较有用）
- `/`：进入搜索模式，继续键入即可过滤列表
- `Enter`：恢复当前选中的会话
- `Esc`：关闭面板；若正在搜索，则先退出搜索，再按一次才关闭
- `Space`：打开当前选中会话的**完整消息记录预览**（用于先看一眼再决定要不要恢复）

搜索会匹配会话标题、首条用户消息、最近助手回复、工作目录，以及该会话关联的模型名等字段。搜不到时会提示 `No sessions match search`。搜索模式下 `Backspace` 可删字，`Ctrl + U` 可清空当前搜索内容。

在预览界面里：

- `Enter`：直接恢复这个会话
- `Esc` 或 `Space`：退回列表
- `↑` / `↓` / `←` / `→`，以及 `h` / `j` / `k` / `l`：翻页查看更早或更晚的内容

## 恢复之后会发生什么

选中并确认后，Hunea 会把当前界面里的对话内容**切换成**该历史会话的可见对话记录内容，并尽量带回当时使用的模型选择，方便你接着聊。成功时一般会有一条类似 `Resumed session <session-id>` 的提示。

需要注意的是：这是**切换到另一条已保存会话**，不是把两条会话的内容合并到一起。当前这条会话本身不会凭空消失，之后再打开 `/resume` 时，它通常也会作为可选项出现在列表里（前提是它已经存储成功，且不再是当前活跃会话）。

如果你只是想清空当前上下文开一个新对话，更适合用 `/clear`（别名 `/new`），而不是 `/resume`。退出应用则见 [`/exit`](/guide/fun/menu/quit)。

## 一点使用建议

- 想快速接上“刚才那一轮”，打开 `/resume` 后直接 `Enter` 第一条通常就够了（列表按最近更新时间倒序，越靠上往往越“新”）。
- 会话多、标题又长时，优先用 `/` 搜索首句用户消息里的关键词，比纯靠眼睛扫列表快很多。
- 不确定是不是那条会话时，先 `Space` 预览完整内容，再 `Enter` 恢复，会比来回切换更省事。
