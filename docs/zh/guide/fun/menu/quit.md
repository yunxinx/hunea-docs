# /exit

`/exit` 用来退出 Hunea 应用。`/quit` 是它的别名，输入 `/exit` 或 `/quit` 效果完全一样。做了别名主要是方便从其他 CLI 产品过渡过来，这两个命名都比较常见。

当你聊完了想正常关掉应用，就可以用它。在空输入框输入 `/`，选中 `/exit` 后按 `Enter` 即可退出。

## 和 Ctrl + C 的区别

也可以用 `Ctrl + C` 退出，但有些不太一样：

- **`/exit` / `/quit`**：确认命令后会**直接退出**，没有二次确认。
- **`Ctrl + C`**：为了避免手滑，默认要**再按一次**才会真正退出。第一次会在状态栏提示 `Press again to exit`，大约 1 秒内再按一次 `Ctrl + C` 才会退出；超时后提示消失，需要重新来一遍。

另外，若输入框里还有未发送的草稿，第一次 `Ctrl + C` 往往会先**清空输入**，并不会立刻进入退出确认；之后再按才会进入上面的二次确认流程。所以：想干净利落地退出，更推荐直接用 `/quit`（或 `/exit`）；`Ctrl + C` 更适合“先清输入 / 再确认退出”的习惯。

## 退出之后

当前版本退出后不会额外打印统计摘要或“如何回到刚才会话”的提示；进程结束后终端会回到你启动 Hunea 之前的状态。

会话本身一般会落在数据目录（通常是 `~/.config/hunea/`）里。下次在同一工作区再启动 Hunea，可以用 [`/resume`](/guide/fun/menu/resume) 从列表里回溯此前的对话，而不是只能从零开始。

如果你只是想**清空当前对话、开一条新会话**，但还不想关掉应用，请使用 `/clear`（别名 `/new`），它与 `/exit` 的用途不同。

## 后续可能的变化

后续可能会在退出时输出一些附加信息，比如类似 Claude Code 或 Codex CLI 那样：

- 提示可以用什么方式快速回到退出前的会话
- 词元（就是 Token 的意思；如未特别指出，当前中文文档里的 Token 都会采用推荐译名 **词元** 来表示）使用量等统计
