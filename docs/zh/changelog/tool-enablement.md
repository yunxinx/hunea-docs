# 工具级启用/禁用

- **日期**：2026-07-16
- **提交**：`feat(prompt): 支持工具级启用/禁用`
- **版本**：`0.11.1-alpha.3`

此前 [`/prompt`](/guide/fun/menu/prompt) 的 Tools 页签只能控制“工具使用指南（guidelines）”是否注入提示词，并不能真正关掉某个工具本身。本次更新让你可以在工具级别决定“这个工具要不要暴露给模型”。

## 主要变化

1. **Tools 页签改为 On/Guide 双列开关**  
   每一行工具现在有两个复选框列：
   - `On`：控制工具本体是否启用。关闭后，该工具不会出现在发送给模型的工具列表里，模型也就无法调用它。
   - `Guide`：控制该工具的使用指南（guidelines）是否注入系统提示词，与旧行为一致。

   On/Guide 双列开关效果如下：

   ![tool-off-on](/assets/changelog/tool-off-on.png)

2. **禁用的工具会从两处一起过滤**  
   关闭 `On` 后，工具会同时从 **provider 请求的工具定义** 与 **会话工具注册表** 中被过滤掉。也就是说，禁用不只是“不告诉模型有这个工具”，而是运行时确实不再提供该工具。

3. **状态会被持久化**  
   工具启停状态（`tool_enablement`）会写入会话存储（SQLite）与项目级 TOML。空会话下改动即时生效；已经开始过的会话仍以“下一次新会话”为主，语义与 `/prompt` 其它改动一致。

## 交互方式

在 [`/prompt`](/guide/fun/menu/prompt) 面板右栏切到 `Tools` 页签后：

| 按键 | 作用 |
| --- | --- |
| `←` / `→` | 在 `On` / `Guide` 两列之间切换（当前列会高亮） |
| `x` | 切换当前列的开关 |

需要注意的联动关系：

- 当某个工具的 `On` 被关闭时，它的 `Guide` 列变为不可操作（弱化显示），此时按 `x` 为无操作；工具本体都禁用了，单独注入它的使用指南没有意义。
- 本身没有 guidelines 的工具，其 `Guide` 列同样是无操作的。

## 和相邻功能的关系

- 内置工具清单、审批策略与路径边界见[工具与审批](/guide/fun/tools)。
- 启停后的最终提示词组装可以在 [`/prompt`](/guide/fun/menu/prompt) 里按 `p` 预览；工具定义占用的上下文见 [`/context`](/guide/fun/menu/context)。
- 多数改动面向“下一次新会话”。想立刻在干净上下文里验证，可在关闭 `/prompt` 后用 `/clear`（别名 `/new`）开一条新会话。
