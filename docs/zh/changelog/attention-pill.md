# 非贴底保留视口与左侧提示

- **日期**：2026-07-17
- **提交**：`feat(tui): 非贴底时保留视口并用左侧 pill 提示`
- **版本**：`0.11.1-alpha.5`

本次调整优化了主会话阅读体验：当你向上滚动查看历史消息时，新内容不会强行将视口拉回底部。

## 贴底跟随逻辑

- 统一了"是否贴底"的判定规则。
- **流式输出**和**运行时刷新**仅在视口确实贴底时才自动跟随到底部。
- 手动滚动离开底部后，视口会保持在当前位置，方便继续阅读上文。

## 左侧提示条

当视口不在底部时，左侧会显示常驻提示条，告知下方有未查看内容：

| 提示 | 含义 |
| --- | --- |
| 新消息计数 | 视口离开底部后，有新的助手消息处理完成 |
| 审批待办 | 工具审批面板已打开但当前不可见（被全屏层遮挡，或因滚动不在可视区域） |

待审批提示条效果如下：

两种提示条都可以**鼠标左键直接点击**，一键回到底部：

- 点击后会先关闭当前遮挡的非审批类全屏层，再将视口跳回底部（恢复贴底跟随），并清除对应提示条。
- 点击**审批待办**提示条时，回到底部后内联审批面板会重新可见并获得输入焦点，方便直接处理。

当视口回到贴底状态，新消息计数会清零；审批提示会在审批处理完成、取消，或面板重新可见时清除。

## 审批面板配套调整

1. **后台打开不抢占全屏**  
   审批可在后台打开，不会强行覆盖当前正在查看的全屏内容。
2. **非贴底状态下开关审批不强制滚动到底部**  
   避免阅读历史时，审批弹窗强行改变视口位置。
3. **面板不可见时，审批快捷键不生效**  
   减少因看不到面板而误审批的风险。

总结：向上滚动阅读可安心进行，有新消息或待审批事项时，左侧会给出提示，不会直接抢占视口。
